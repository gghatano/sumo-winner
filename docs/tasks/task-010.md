# Task 010: スクレイパの最終フォールバックで幕内以外を取り込む問題を修正

## 優先度
**修正必須**

## 問題
`scripts/fetch_torikumi.py:154` - `parse_torikumi` の最終フォールバックが全テーブル走査のため、「幕内」セクションが見つからない場合に十両等を誤って取り込む可能性がある。仕様の「幕内のみ」に違反。

## ブランチ / worktree
- `gitworktree/feature-010-scraper-makuuchi-only`

## 依存タスク
- なし

## 作業内容

1. **`scripts/fetch_torikumi.py` の `parse_torikumi` を修正**
   - 最終フォールバック（全テーブル走査）を削除または制限
   - 幕内のコンテキストが確認できない場合は空配列を返す
   - ログで「幕内セクションが見つかりませんでした」と警告を出力

2. **テスト追加** (`tests/test_parser.py`)
   - 幕内セクションが存在しないHTMLを入力した場合、空配列が返ることを検証
   - 既存テスト `test_empty_html` のアサーションを `== []` に変更（`isinstance` だけでなく）

## 完了条件
- 幕内セクションが見つからない場合は空配列を返す
- 十両のみのHTMLを入力しても十両の取組を返さない
- テストが全パスする
